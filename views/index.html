Check Box

체크 박스는 목록 중 선택할 항목이 여러개 있을때 사용합니다.

  • 체크박스는 2개 크기로 구분합니다.
  • 체크박스는 default, hover, checked, indeterminate, disabled 총 5개 상태를 갖습니다.
  • 체크박스를 하나만 사용하는 경우 toggle-switch를 사용해 두 상태를 전환하는 것과 같습니다.
    두개의 차이는 Switch는 상태 변경이 직접 반영되며, Checkbox는 상태 변경 후 Submit 동작을 수행해야 반영됩니다.

Select

특정 버튼을 클릭했을 경우 뜨는 리스트 형태의 메뉴입니다.

  • 셀렉트의 유형이 2개 있습니다.
    • 텍스트 / 아이콘 + 텍스트 / 텍스트 + 서브텍스트
    • 멀티 셀렉트 - 다중 선택이 가능한 UI
    • 콤보 박스 - 입력 필드와 리스트가 결합한 UI
  • 셀렉트의 테마가 2개 있습니다.
    • select - 보더가 있는 UI
    • select--borderless - 보더가 없는 UI
  • 셀렉트의 사이즈가 3개 있습니다.
    • select--lg / select / select--sm
  • 셀렉트의 리스트의 방향을 설정할 수 있습니다.
    • 기본 - 셀렉트 버튼을 기준으로 좌측 하단으로 펼쳐지는 UI
    • select--left - 셀렉트 버튼을 기준으로 좌측에서부터 셀렉트 리스트가 나열되는 UI
    • select--up - 셀렉트 버튼을 기준 위로 셀렉트 리스트가 펼쳐지는 UI
    • select--right - 셀렉트 버튼을 기준으로 우측에서부터 셀렉트 리스트가 나열되는 UI
  • select--view-all 클래스를 추가하면 셀렉트 리스트가 텍스트 길이에 따라 계속해서 늘어납니다.

Tag

태그는 각 항목을 설명하는 키워드로 사용하여
항목에 레이블 지정, 분류(categorizing) 또는 정리할 수 있습니다.

  • 태그는 2개 유형, 2개 스타일이 있으며 3개 크기로 구분합니다.
  • 유형: Basic Tag 유형과 제거 버튼이 있는 Removable Tag 2개 유형
  • 테마: 전체 서비스에 사용하는 Default 테마와 Studio에만 사용하는 2개 테마
  • 태그를 한번 클릭하여 토글할 수 있습니다. tag--selected 클래스 추가/제거로 제어합니다.
  • Studio 테마는 엘리먼트 상위에 canvas-object 클래스 여부에 따라 적용됩니다.
